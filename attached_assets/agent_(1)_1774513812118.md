# WEBSPACEAI Dataset Engine — Agent.md (Engineering Spec v2)

## 0. Executive Summary
This document defines a production-oriented, modular dataset factory for WEBSPACEAI. The system ingests heterogeneous sources, normalizes and segments content, synthesizes training examples, validates quality, versions outputs, and exposes datasets for RAG, evaluation, and fine-tuning.

**North Star**: deterministic, observable pipelines that continuously improve data quality with minimal human intervention.

---

## 1. System Principles
- **Determinism first**: same inputs + config ⇒ same outputs (seeded generation where possible)
- **Observability over magic**: every stage logs inputs/outputs + metrics
- **Schema rigidity**: strict contracts between stages (pydantic/jsonschema)
- **Idempotency**: stages can be retried safely
- **Separation of concerns**: ingestion ≠ cleaning ≠ generation ≠ validation
- **Config-driven**: pipelines defined via versioned configs (YAML/JSON)
- **Fail fast, degrade gracefully**: partial outputs allowed with clear flags

---

## 2. High-Level Dataflow
```
Sources → Ingest → Normalize → Deduplicate → Chunk → Generate → Validate → Score → Filter → Version → Export
```

Each arrow is a **materialized boundary** with persisted artifacts for replay/debug.

## 5. Pipeline Modules (Deep Spec)

### 5.1 Ingest
- URL fetch (timeout, retries, user-agent)
- HTML → text extraction (trafilatura primary, bs4 fallback)
- File parsers (PDF via PyMuPDF, TXT, MD)
- Normalize encoding (UTF-8)
- **Output**: `raw/*.jsonl`

### 5.2 Normalize & Clean
- boilerplate removal (nav/footer heuristics)
- whitespace normalization
- sentence boundary detection
- language detection (fastText optional)
- **Output**: `processed/*.jsonl`

### 5.3 Deduplication
- exact hash (sha256)
- near-dup via shingling + MinHash or rapidfuzz similarity
- maintain `seen_hashes`
- **Output**: `dedup/*.jsonl`

### 5.4 Chunking
- sliding window (target 600 tokens, overlap 80)
- semantic hints: split on headings, paragraphs
- compute token estimate via tiktoken
- attach `section`
- **Output**: `chunks/*.jsonl`

### 5.5 Generation (LLM-backed)
Modes:
- instruction
- qa
- summary
- chat (2–4 turns)

**Prompt contract**:
- include chunk text
- forbid hallucination beyond chunk
- require JSON output only

**Determinism**:
- set temperature low (0–0.3)
- include seed when supported

**Output**: `generated/*.jsonl`

### 5.6 Validation
- JSON schema validation
- min/max length checks
- grounding check: n-gram overlap with source
- duplication check across dataset (hash of normalized fields)
- safety filters (keyword rules)

### 5.7 Scoring
Heuristics + LLM scorer (optional):
- relevance: overlap with source
- clarity: length + punctuation + readability
- grounding: extractive overlap ratio
- diversity: similarity against existing pool

`final = weighted_sum(relevance, grounding, clarity, diversity)`

### 5.8 Filtering
- drop if `final < threshold` (e.g., 0.75)
- cap per-source contribution to avoid bias

### 5.9 Versioning
- each run has `run_id`
- config snapshot saved
- artifacts stored under `/versions/{run_id}/`

### 5.10 Export
- JSONL (primary)
- CSV (inspection)
- Report (Markdown): counts, score distribution, drop rates

  score_threshold: 0.75

  max_records: 5000
